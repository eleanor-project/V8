# Documentation

Start with `docs/index.md` for the documentation hub and navigation map.

## Structure

- `docs/architecture/` - system design and diagrams
- `docs/api-reference/` - public APIs and protocols
- `docs/guides/` - onboarding and usage
- `docs/integration/` - integration recipes
- `docs/concepts/` - deep dives on core concepts
- `docs/development/` - contribution and release process
- `docs/examples/` - runnable example walkthroughs
- `docs/reference/` - configuration, errors, metrics, changelog
